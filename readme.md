Пояснительная записка к тестовому заданию.

В качестве ORM используется Prisma.
Аутентификация выполнена на JWT.

Метод получения пользователя по ID после раздумий был разделен на 2 отдельных метода:
- метод getMe для всех авторизованных (имеют токен)
- и метод getUserById (только для администратора)

Если делать это одним методом (например ../getUserById/1), то пользователь обязан на бекенд передать свой уникальный идентификатор (ID), который он знать не может.

Метод блокировки был разделен также на два метода - один для залогиненного пользователя, другой для администратора

Что не сделано:
- интерфейсы не вынесены в отдельный файл
- по умолчанию возвращаются все поля модели "user", включая хеш пароля
- нет проверки на блокировку пользователя
- нет учета выданных токенов
- запросы на бекенд не оптимизированы
